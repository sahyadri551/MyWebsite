JavaScript Functions

Definition

A function is a reusable block of code designed to perform a particular task. Functions are the fundamental building blocks of JavaScript programs. They allow you to organize your code, make it more readable, and avoid repetition. In JavaScript, functions are "first-class citizens," which means they can be treated like any other value: assigned to variables, passed as arguments to other functions, and returned by other functions.

Function Basics

Declaration, Expression, and Calling

There are two primary ways to create a function:

  • Function Declaration: A named function defined using the `function` keyword. Declarations are "hoisted," meaning they are available throughout their scope even if called before they are defined.
  • Function Expression: A function created and assigned to a variable. Expressions are not hoisted, so they can only be called after they are defined.
// Function Declaration
function greet(name) {
  return `Hello, ${name}!`;
}

// Function Expression
const add = function(a, b) {
  return a + b;
};

// Calling a function
console.log(greet('Alice')); // Output: Hello, Alice!
console.log(add(5, 10)); // Output: 15

Parameters, Arguments, and Default Arguments

  • Parameters: The names listed in the function definition (e.g., `name`, `a`, `b` above).
  • Arguments: The real values passed to the function when it is called (e.g., `'Alice'`, `5`, `10` above).
  • Default Arguments (ES6): You can provide default values for parameters, which are used if an argument is not provided.
function sayHello(name = 'Guest') {
  console.log(`Hello, ${name}!`);
}
sayHello('Bob'); // Output: Hello, Bob!
sayHello();     // Output: Hello, Guest!

The `return` Statement

The `return` statement ends function execution and specifies a value to be returned to the function caller. If omitted, a function implicitly returns `undefined`.

Function Scope

  • Global Scope: Variables declared outside any function are in the global scope and can be accessed from anywhere.
  • Local Scope (Function Scope): Variables declared inside a function are in a local scope and can only be accessed within that function.
  • Block Scope: With `let` and `const`, variables are scoped to the nearest block (`{...}`), such as an `if` statement or a `for` loop.

Intermediate Function Topics

Arrow Functions (ES6)

A concise syntax for writing function expressions. They are anonymous and lexically bind the `this` value, which is a key difference from regular functions.

// Regular function expression
const multiply = function(a, b) {
  return a * b;
};

// Arrow function
const multiplyArrow = (a, b) => a * b; // Implicit return

Immediately Invoked Function Expressions (IIFE)

An IIFE is a function that is defined and executed immediately. It's a common pattern for creating a local scope to avoid polluting the global namespace.

(function() {
  let message = 'This is private.';
  console.log(message);
})();
// The 'message' variable is not accessible globally.

Callback and Higher-Order Functions

  • Callback Function: A function passed as an argument to another function, to be "called back" at a later time.
  • Higher-Order Function: A function that either takes one or more functions as arguments, or returns a function as its result.
// `calculator` is a higher-order function
// `operation` is a callback function
function calculator(a, b, operation) {
  return operation(a, b);
}

const sum = (x, y) => x + y;
const product = (x, y) => x * y;

console.log(calculator(10, 5, sum));     // 15
console.log(calculator(10, 5, product)); // 50

Rest Parameters and Spread Syntax

Both use the `...` syntax but for opposite purposes.

  • Rest Parameters: Collects an indefinite number of arguments into an array.
  • Spread Syntax: Expands an iterable (like an array) into individual elements.
// Rest Parameters
function sumAll(...numbers) {
  return numbers.reduce((acc, curr) => acc + curr, 0);
}
console.log(sumAll(1, 2, 3, 4)); // 10

// Spread Syntax
const arr1 = [1, 2, 3];
const arr2 = [...arr1, 4, 5];
console.log(arr2); // [1, 2, 3, 4, 5]

Advanced Function Topics

Closures

A closure is a function that "remembers" the environment in which it was created. This means it has access to variables from its outer (enclosing) function, even after the outer function has finished executing. Closures are a powerful feature for creating private data and stateful functions.

function createCounter() {
  let count = 0; // This variable is in the closure

  return function() {
    count++;
    return count;
  };
}

const counter1 = createCounter();
console.log(counter1()); // 1
console.log(counter1()); // 2

const counter2 = createCounter(); // Has its own separate `count`
console.log(counter2()); // 1

The `this` Keyword

The value of `this` is determined by how a function is called (its "call site").

  • Global Context: In the global scope, `this` refers to the global object (`window` in browsers).
  • Inside a Method: When a function is called as a method of an object (`obj.myMethod()`), `this` refers to the object itself.
  • Arrow Functions: They do not have their own `this` binding. Instead, they inherit `this` from the parent scope (lexical `this`). This is extremely useful for callbacks inside methods.

Currying

Currying is the process of transforming a function that takes multiple arguments into a sequence of nested functions, each taking a single argument.

// A non-curried function
const add = (a, b, c) => a + b + c;

// The same function, curried
const curriedAdd = a => b => c => a + b + c;

console.log(curriedAdd(2)(3)(5)); // 10

const add5 = curriedAdd(5); // Create a specialized function
console.log(add5(10)(1)); // 16

Async Functions (`async`/`await`)

The `async` and `await` keywords provide syntactic sugar over Promises, making asynchronous code look and behave more like synchronous code, which is easier to read and debug.

  • An `async` function always returns a Promise.
  • The `await` keyword can only be used inside an `async` function. It pauses the function's execution and waits for a Promise to resolve before continuing.
async function getUserData(userId) {
  try {
    const response = await fetch(`https://api.example.com/users/${userId}`);
    const user = await response.json();
    console.log(user);
  } catch (error) {
    console.error('Failed to fetch user:', error);
  }
}
