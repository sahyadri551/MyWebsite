JavaScript Loops

Introduction

Loops are a fundamental concept in programming that allow you to execute a block of code repeatedly. They are used to automate repetitive tasks, iterate over collections of data like arrays and objects, and perform calculations until a certain condition is met. Without loops, you would have to write the same code over and over again, making your programs verbose and difficult to maintain.

Types of Loops

JavaScript provides several ways to perform iteration. The main types of loops are:

  • for loop: The most common loop, used when you know the number of iterations beforehand.
  • while loop: Executes a block of code as long as a specified condition is true.
  • do...while loop: Similar to the while loop, but it executes the code block at least once before checking the condition.
  • for...of loop: Iterates over the values of an iterable object (like Arrays, Strings, Maps, Sets).
  • for...in loop: Iterates over the enumerable properties (keys) of an object.

Standard Loops Explained

The for Loop

The for loop is ideal when you know how many times you want the loop to run. It consists of three parts: initialization, condition, and final expression (increment/decrement).

Syntax:

for (initialization; condition; finalExpression) {
  // code to be executed
}

Example:

for (let i = 0; i < 5; i++) {
  console.log(`The number is ${i}`);
}

Result:

The number is 0
The number is 1
The number is 2
The number is 3
The number is 4

The while Loop

The while loop continues to execute as long as the specified condition evaluates to true. The condition is checked before each iteration.

Syntax:

while (condition) {
  // code to be executed
}

Example:

let count = 0;
while (count < 3) {
  console.log(`Count is ${count}`);
  count++;
}

Result:

Count is 0
Count is 1
Count is 2

The do...while Loop

This loop is a variant of the while loop. The key difference is that the code block is executed at least once, and the condition is checked after the execution.

Syntax:

do {
  // code to be executed
} while (condition);

Example:

let i = 5;
do {
  console.log(`The value is ${i}`);
  i++;
} while (i < 5);

Result:

The value is 5

Array and Object Loops

The for...of Loop

Introduced in ES6, the for...of loop is the modern way to iterate over iterable objects like arrays, strings, maps, and sets. It directly gives you the value of each element.

Syntax:

for (const element of iterable) {
  // code to be executed with element
}

Example:

const fruits = ['Apple', 'Banana', 'Cherry'];
for (const fruit of fruits) {
  console.log(fruit);
}

Result:

Apple
Banana
Cherry

The for...in Loop

The for...in loop iterates over the enumerable property keys (or indices) of an object. It's generally used for plain objects, not for arrays, as it can have unexpected behavior with array inheritance and order.

Syntax:

for (const key in object) {
  // code to be executed with key
}

Example:

const user = {
  name: 'Alice',
  age: 30,
  isAdmin: true
};
for (const key in user) {
  console.log(`${key}: ${user[key]}`);
}

Result:

name: Alice
age: 30
isAdmin: true

Loop Control Statements

Control statements change the normal execution flow of a loop.

  • break: Immediately terminates the current loop and transfers control to the statement following the loop.
  • continue: Skips the rest of the current iteration and proceeds to the next one.

Example (break):

for (let i = 0; i < 10; i++) {
  if (i === 3) {
    break; // Stop the loop when i is 3
  }
  console.log(i);
}
// Result: 0, 1, 2

Example (continue):

for (let i = 0; i < 5; i++) {
  if (i === 2) {
    continue; // Skip the rest of this iteration when i is 2
  }
  console.log(i);
}
// Result: 0, 1, 3, 4

Higher-Order Array Methods

Arrays in JavaScript come with powerful, built-in methods that use functions to perform iteration and are often more concise and readable than traditional loops.

.forEach()

Executes a provided function once for each array element. It's a modern replacement for a simple for loop.

const colors = ['red', 'green', 'blue'];
colors.forEach((color, index) => {
  console.log(`Color at index ${index} is ${color}`);
});
// Result: Logs each color with its index.

.map()

Creates a new array populated with the results of calling a provided function on every element in the calling array.

const numbers = [1, 2, 3];
const doubled = numbers.map(num => num * 2);
console.log(doubled);
// Result: [2, 4, 6]

.filter()

Creates a new array with all elements that pass the test implemented by the provided function.

const ages = [15, 20, 35, 18];
const adults = ages.filter(age => age >= 18);
console.log(adults);
// Result: [20, 35, 18]

.reduce()

Executes a reducer function on each element of the array, resulting in a single output value.

const values = [1, 2, 3, 4];
const sum = values.reduce((accumulator, currentValue) => accumulator + currentValue, 0);
console.log(sum);
// Result: 10

.some() and .every()

These methods test whether elements in the array pass a test. They return a boolean.

  • .some(): Returns true if at least one element passes the test.
  • .every(): Returns true only if all elements pass the test.
const scores = [85, 92, 78, 40, 95];
const hasFailingScore = scores.some(score => score < 50); // true
const allPassingScores = scores.every(score => score >= 50); // false

.find() and .findIndex()

These methods find an element in an array.

  • .find(): Returns the first element that satisfies the provided testing function. Otherwise undefined is returned.
  • .findIndex(): Returns the index of the first element that satisfies the test. Otherwise -1 is returned.
const products = [
  { id: 'a', name: 'Laptop' },
  { id: 'b', name: 'Mouse' },
  { id: 'c', name: 'Keyboard' }
];
const mouse = products.find(p => p.name === 'Mouse');
console.log(mouse); // { id: 'b', name: 'Mouse' }

const mouseIndex = products.findIndex(p => p.name === 'Mouse');
console.log(mouseIndex); // 1

Advanced Loop Topics

Loop performance and optimization

For most applications, loop performance is not a bottleneck. However, in performance-critical code, consider these tips:

  • Choosing the right loop: A classic for loop with a cached array length (for (let i = 0, len = arr.length; i < len; i++)) can be marginally faster than .forEach() or for...of for extremely large arrays because it has less overhead.
  • Minimize work inside the loop: Avoid complex calculations, function calls, or DOM manipulation inside a loop if they can be done outside.

Async/Await inside loops

Using await inside loops is a common requirement. The for...of loop is the best choice as it correctly handles await and pauses the loop until the promise resolves.

Note: You cannot use await inside .forEach() because it does not wait for the async operations to complete.

async function processUsers(userIds) {
  console.log('Processing users...');
  for (const id of userIds) {
    const user = await fetchUserData(id); // Loop pauses here
    console.log(`Processed ${user.name}`);
  }
  console.log('All users processed.');
}

Nested loops

A nested loop is a loop inside another loop. This is common for working with multi-dimensional arrays or creating combinations of data. Be mindful of performance, as the complexity is often O(n2) or higher.

const matrix = [[1, 2], [3, 4]];
for (let i = 0; i < matrix.length; i++) {
  for (let j = 0; j < matrix[i].length; j++) {
    console.log(matrix[i][j]);
  }
}
// Result: 1, 2, 3, 4

Looping over Sets and Maps

Both Set and Map are iterable, so for...of and .forEach() work perfectly.

// Set
const mySet = new Set(['a', 'b', 'c']);
for (const value of mySet) {
  console.log(value);
}

// Map
const myMap = new Map([ ['key1', 'value1'], ['key2', 'value2'] ]);
myMap.forEach((value, key) => {
  console.log(`${key} = ${value}`);
});

Infinite loops and how to avoid them

An infinite loop occurs when the loop's condition never becomes false. This will freeze the browser or server.

Cause: Usually happens when the counter variable is not updated correctly, or the condition is always true.

// DANGER: Infinite Loop! The condition i < 5 is always true.
for (let i = 0; i < 5;) {
  console.log('This will run forever');
}

Prevention: Always ensure that the loop's condition will eventually be met. Double-check your update expressions (e.g., i++) and your conditional logic.

Recursion as an alternative to loops

Recursion is a technique where a function calls itself to solve a problem. Any problem solvable with a loop can also be solved with recursion. It's often used for tasks involving tree-like data structures.

Example (Factorial):

// Using a loop
function factorialLoop(n) {
  let result = 1;
  for (let i = n; i > 1; i--) {
    result *= i;
  }
  return result;
}

// Using recursion
function factorialRecursive(n) {
  if (n <= 1) { // Base case: stops the recursion
    return 1;
  }
  return n * factorialRecursive(n - 1); // Recursive call
}
