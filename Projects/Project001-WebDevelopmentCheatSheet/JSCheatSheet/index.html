JavaScript Intro

JavaScript is a high-level, interpreted programming language that conforms to the ECMAScript specification. It is a language that is also characterized as dynamic, weakly typed, prototype-based and multi-paradigm. Alongside HTML and CSS, JavaScript is one of the core technologies of the World Wide Web. It enables interactive web pages and is an essential part of web applications. The vast majority of websites use it, and all major web browsers have a dedicated JavaScript engine to execute it.

JS History

Created in 10 days in May 1995 by Brendan Eich, then a programmer at Netscape, JavaScript was initially named Mocha. It was later renamed to LiveScript and finally to JavaScript. The name change was a marketing move to capitalize on the popularity of Sun Microsystems' Java language, despite the two having very little in common. In 1997, JavaScript was standardized by ECMA International as ECMAScript.

ES5 and ES6 Updates

ES5 (ECMAScript 5), published in 2009, was the first major update to the language in a decade. It introduced several new features and improvements, including:

  • use strict directive for writing safer code.
  • Array methods like forEach, map, filter, reduce, and reduceRight.
  • JSON.parse() and JSON.stringify() for working with JSON data.
  • Enhancements to the Object constructor, including Object.keys(), Object.values(), Object.entries(), Object.assign(), and other object-related enhancements.

ES6 (ECMAScript 2015), also known as ECMAScript 2015, was a significant and highly anticipated update that brought modern syntax and features to the language:

  • let and const for block-scoped variables.
  • Arrow functions for a more concise function syntax.
  • Classes for a more familiar object-oriented programming syntax.
  • Template literals for easier string interpolation.
  • Destructuring assignment for arrays and objects.
  • Default function parameters.
  • Promises for better asynchronous programming.

JS Features

  • Lightweight and Interpreted: JavaScript is a lightweight language, and web browsers interpret it directly without the need for compilation.
  • First-Class Functions: Functions in JavaScript are treated as first-class citizens, meaning they can be assigned to variables, passed as arguments to other functions, and returned from functions.
  • Prototype-Based: JavaScript uses prototypes for inheritance rather than classes (though ES6 introduced class syntax, it's syntactic sugar over the existing prototype-based inheritance).
  • Dynamic Typing: A variable in JavaScript can hold values of different data types during the execution of a program.
  • Event-Driven and Asynchronous: JavaScript is well-suited for handling user interactions and asynchronous operations, making it ideal for building responsive user interfaces.
  • Platform Independent: JavaScript can run on any platform with a JavaScript engine, making it a highly portable language.

Popular JS Frontend and Backend Frameworks

Frontend Frameworks

  • React: A JavaScript library for building user interfaces, developed and maintained by Facebook. It is known for its component-based architecture and virtual DOM.
  • Angular: A TypeScript-based open-source web application framework led by the Angular Team at Google. It is a complete rewrite of AngularJS and is known for its comprehensive features.
  • Vue.js: A progressive JavaScript framework for building user interfaces. It is known for its approachability, versatility, and performance.

Backend Frameworks

  • Node.js: A JavaScript runtime built on Chrome's V8 JavaScript engine. It allows developers to run JavaScript on the server-side.
  • Express.js: A minimal and flexible Node.js web application framework that provides a robust set of features for web and mobile applications.
  • Next.js: A React framework for production that enables functionalities like server-side rendering and generating static websites.

JS Engine Structure And Working

A JavaScript engine is a computer program that executes JavaScript code. The first JavaScript engines were mere interpreters, but all modern engines use just-in-time (JIT) compilation for improved performance.

Structure

A typical JavaScript engine consists of two main components:

  • Memory Heap: This is where memory allocation happens. Objects are stored in the heap.
  • Call Stack: This is where the engine keeps track of the functions that are being executed. When a script calls a function, it's added to the top of the call stack. When the function returns, it's popped off the stack.

Working

The process of executing JavaScript code involves several steps:

  1. Parsing: The engine reads the source code and converts it into an Abstract Syntax Tree (AST).
  2. Compilation: The AST is then passed to the engine's interpreter or JIT compiler. The interpreter, like Ignition in V8, generates bytecode from the AST.
  3. Optimization: The engine's optimizing compiler, like TurboFan in V8, takes the bytecode and profiling data and produces highly optimized machine code. If the profiling data suggests an optimization was incorrect, the engine de-optimizes and falls back to the interpreter's bytecode.
  4. Execution: The optimized machine code is then executed.
