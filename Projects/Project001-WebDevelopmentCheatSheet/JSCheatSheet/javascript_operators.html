JavaScript Operators

Introduction

In JavaScript, an operator is a special symbol used to perform operations on operands (values and variables). For example, in 10 + 5, the + sign is an operator and 10 and 5 are operands. Operators are the building blocks of any program, allowing you to manipulate data, compare values, and combine multiple conditions.

Types of Operators by Number of Operands

Operators can be classified based on the number of operands they work with.

  • Unary: A unary operator works on a single operand. 
    let x = 5;
x++; // '++' is a unary operator
console.log(!true); // '!' is a unary operator
  • Binary: A binary operator works on two operands. This is the most common type. 
    let sum = 10 + 20; // '+' is a binary operator
  • Ternary: A ternary operator is the only operator in JavaScript that works on three operands. 
    let result = (10 > 5) ? 'Yes' : 'No'; // '?:' is a ternary operator

Types of Operators by Usage

Arithmetic Operators

Used to perform mathematical calculations.

  • + (Addition), - (Subtraction), * (Multiplication), / (Division)
  • % (Modulus - remainder of division)
  • ** (Exponentiation - ES6)
  • ++ (Increment), -- (Decrement)
let a = 10;
let b = 4;
console.log(a + b);  // 14
console.log(a % b);   // 2
console.log(2 ** 3); // 8
a++;
console.log(a);      // 11

Assignment Operators

Used to assign values to variables.

  • = (Assignment)
  • +=, -=, *=, /=, %=, **= (Compound Assignment)
let x = 10;
x += 5; // equivalent to x = x + 5
console.log(x); // 15

Comparison Operators

Used to compare two values, resulting in a Boolean value (true or false).

  • == (Equal to), != (Not equal to)
  • === (Strictly equal to - value and type), !== (Strictly not equal to)
  • > (Greater than), < (Less than), >= (Greater than or equal to), <= (Less than or equal to)
let age = 25;
console.log(age > 18);    // true
console.log('25' == 25);  // true (loose equality)
console.log('25' === 25); // false (strict equality)

Logical Operators

Used to combine or invert Boolean values.

  • && (Logical AND)
  • || (Logical OR)
  • ! (Logical NOT)
let isAdult = true;
let hasLicense = false;
console.log(isAdult && hasLicense); // false
console.log(isAdult || hasLicense); // true
console.log(!isAdult);              // false

Bitwise Operators

Treat operands as 32-bit signed integers and perform operations on their binary representation.

  • & (AND), | (OR), ^ (XOR), ~ (NOT)
  • << (Left shift), >> (Sign-propagating right shift), >>> (Zero-fill right shift)
// 5 = 0101, 1 = 0001
console.log(5 & 1); // 1 (0001)
console.log(5 | 1); // 5 (0101)
console.log(~5);    // -6 (Explained below)

Ternary (Conditional) Operator

A shortcut for an if...else statement.

let temperature = 25;
let weather = (temperature > 20) ? 'Warm' : 'Cool';
console.log(weather); // Warm

Differentiate between == and ===

Understanding the difference between the loose equality (==) and strict equality (===) operators is crucial in JavaScript.

Feature == (Loose Equality) === (Strict Equality)
Comparison Compares only the values of the operands. Compares both the values and the data types of the operands.
Type Coercion Performs type coercion, meaning it converts operands to the same type before comparison. Does not perform type coercion. If the types are different, it immediately returns false.
Usage Can lead to unexpected results due to automatic type conversion. Generally advised against. Predictable and safer. It is the recommended equality operator.

Example

console.log(77 == '77');  // true, because '77' is converted to the number 77 before comparison.
console.log(77 === '77'); // false, because one is a number and the other is a string.

console.log(true == 1);   // true, because true is converted to the number 1.
console.log(true === 1);  // false, because one is a boolean and the other is a number.

console.log(null == undefined);   // true, a special case in the equality algorithm.
console.log(null === undefined);  // false, because they are of different types.

Explain ~ (Bitwise NOT) Operator Clearly

The Bitwise NOT (~) operator inverts all the bits of its operand. It operates on a 32-bit signed integer. The process can be confusing because of how negative numbers are stored in binary using a format called Two's Complement.

The formula for the result of ~n is simply -(n + 1).

Let's see why ~5 equals -6:

  1. Convert to 32-bit Binary: The number 5 in a 32-bit representation is: 
    00000000 00000000 00000000 00000101
  2. Invert all bits: The ~ operator flips every 0 to a 1 and every 1 to a 0. 
    11111111 11111111 11111111 11111010
  3. Interpret as a Signed Integer: Since the first bit (the sign bit) is 1, the number is negative. To find its decimal value, the computer uses the Two's Complement method:
    • Invert all the bits again: 00000000 00000000 00000000 00000101
    • Add 1: 00000000 00000000 00000000 00000110
  4. Result: The resulting binary is the representation for the number 6. Since the sign bit was 1, the final decimal value is -6.

typeof and instanceof Operators

typeof Operator

The typeof operator is a unary operator that returns a string indicating the data type of its operand.

console.log(typeof 42);             // "number"
console.log(typeof 'hello');        // "string"
console.log(typeof true);           // "boolean"
console.log(typeof undefined);      // "undefined"
console.log(typeof { a: 1 });       // "object"
console.log(typeof [1, 2, 3]);      // "object" (arrays are objects)
console.log(typeof null);           // "object" (this is a well-known historical bug)
console.log(typeof function() {});  // "function"
console.log(typeof Symbol('id'));   // "symbol"

instanceof Operator

The instanceof operator checks if an object is an instance of a particular constructor or class. It tests whether the prototype property of a constructor appears anywhere in the prototype chain of an object.

class Car {}
let myCar = new Car();

console.log(myCar instanceof Car);        // true

let colors = ['red', 'green', 'blue'];
console.log(colors instanceof Array);     // true
console.log(colors instanceof Object);    // true (because Array inherits from Object)

let date = new Date();
console.log(date instanceof Date);        // true

console.log('Hello' instanceof String);   // false (a primitive is not an instance)
let strObj = new String('Hello');
console.log(strObj instanceof String);  // true (this is a String object)

Nullish Coalescing Operator (??)

The nullish coalescing operator (??) is a logical operator that returns its right-hand side operand when its left-hand side operand is null or undefined, and otherwise returns its left-hand side operand.

It's a way to provide a default value for a variable only when it is "nullish" (i.e., null or undefined).

Usage and Example

This is particularly useful because it treats other "falsy" values like 0, '' (empty string), and false as valid values, unlike the logical OR (||) operator.

// Using || (Logical OR)
let quantity = 0;
let displayQuantity_OR = quantity || 10;
console.log(displayQuantity_OR); // 10 (because 0 is a falsy value, || falls back to the default)

// Using ?? (Nullish Coalescing)
let displayQuantity_NC = quantity ?? 10;
console.log(displayQuantity_NC); // 0 (because 0 is not null or undefined, it is considered a valid value)

let user; // undefined
let defaultUser = 'Guest';
console.log(user ?? defaultUser); // 'Guest'

let config = { timeout: 0, retries: null };
let timeout = config.timeout ?? 5000;   // timeout is 0
let retries = config.retries ?? 3;       // retries is 3
let host = config.host ?? 'localhost';   // host is 'localhost'
