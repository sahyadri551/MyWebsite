Definition

In programming, a data type is a classification that specifies which type of value a variable has and what type of mathematical, relational, or logical operations can be applied to it without causing an error. JavaScript is a dynamically typed language, which means you don't have to declare the data type of a variable. The JavaScript engine determines the data type of a variable at runtime.

Detailed classification of primitive and non-primitive data types

JavaScript data types are divided into two main categories:

  • Primitive Data Types: These are the most basic data types in JavaScript. They are immutable, meaning their values cannot be changed once created. They are stored directly in the location that the variable accesses. There are seven primitive data types: String, Number, BigInt, Boolean, Undefined, Null, and Symbol.
  • Non-Primitive Data Types (Reference Types): These data types can hold collections of values and more complex entities. They are mutable, and their values are stored as a reference in memory. The primary non-primitive data type is the Object, which includes special object types like Array and Function.

Explain Primitive Datatypes With proper example.

String

The String data type is used to represent textual data. It is a sequence of characters enclosed in single quotes (`'`) or double quotes (`"`).

let greeting = "Hello, World!";
let name = 'Alice';

Number

The Number data type represents both integer and floating-point numbers. It also includes special values like `Infinity`, `-Infinity`, and `NaN` (Not-a-Number).

let age = 30;
let price = 19.99;
let temperature = -5.5;

Boolean

The Boolean data type represents a logical entity and can have one of two values: `true` or `false`.

let isLoggedIn = true;
let hasPermission = false;

Undefined

A variable that has not been assigned a value has the value `undefined`. It is a primitive value automatically assigned to variables that have just been declared.

let score;
console.log(score); // Outputs: undefined

Null

The Null type has exactly one value: `null`. It represents the intentional absence of any object value. It is often used to signify that a variable has no value.

let user = null;

Symbol

Introduced in ES6, a Symbol is a unique and immutable primitive value that may be used as the key of an Object property. Every Symbol value returned from `Symbol()` is unique.

const id1 = Symbol('id');
const id2 = Symbol('id');
console.log(id1 === id2); // Outputs: false

BigInt

BigInt is a numeric primitive in JavaScript that can represent integers with arbitrary precision. BigInts are created by appending `n` to the end of an integer or by calling the `BigInt()` constructor.

const bigNumber = 9007199254740991n;
const anotherBigNumber = BigInt("9007199254740992");

Differentiate between null and undefined with some examples.

While both `null` and `undefined` represent the absence of a value, they are used in different contexts.

Feature Undefined Null
Meaning A variable has been declared but has not been assigned a value. An intentional absence of any object value. It is an assignment value.
Type `typeof undefined` is `"undefined"`. `typeof null` is `"object"` (this is a long-standing bug in JavaScript).
Example `let x; console.log(x); // undefined` `let y = null; console.log(y); // null`
Comparison `null == undefined` is `true` (loose equality). `null === undefined` is `false` (strict equality).

Examples:

let a;
console.log(a); // undefined

function greet(name) {
    console.log(`Hello, ${name}`);
}
greet(); // Hello, undefined

let b = null;
console.log(b); // null

let obj = { name: "John", age: null };
console.log(obj.age); // null (intentionally set to no value)

Explain Mutable and immutable and differentiate between them.

Immutable data types are those whose values cannot be changed after they are created. When you perform an operation that seems to modify an immutable value, you are actually creating a new value in memory. All primitive data types in JavaScript are immutable.

Mutable data types are those whose values can be changed in place, after they are created. Non-primitive data types (objects, arrays) are mutable.

Example with an Immutable (Primitive) Type:

let str = "hello";
str[0] = "H"; // This will not cause an error, but it won't change the string
console.log(str); // Outputs: "hello"

let newStr = str.toUpperCase();
console.log(newStr); // Outputs: "HELLO"
console.log(str); // Outputs: "hello" (original string is unchanged)

Example with a Mutable (Non-Primitive) Type:

let person = { name: "John", age: 30 };
person.age = 31;
console.log(person); // Outputs: { name: "John", age: 31 }

const colors = ["red", "green"];
colors.push("blue");
console.log(colors); // Outputs: ["red", "green", "blue"]
