JavaScript Objects, Sets & Maps

Objects

An **object** is a fundamental data structure in JavaScript. It's a collection of key-value pairs, where keys are usually strings (or Symbols) and values can be any data type, including other objects or functions. Objects are used to group related data and functionality.

Creating and Accessing Objects

The most common way to create an object is with an object literal (`{}`). You can access and modify properties using dot notation (`.`) or bracket notation (`[]`). Bracket notation is required when the key is a variable or contains special characters.

// Object literal
const user = {
  firstName: 'John',
  lastName: 'Doe',
  age: 30,
  'has-admin-access': true,
  sayHello: function() {
    console.log(`Hello, my name is ${this.firstName}`);
  }
};

// Accessing properties
console.log(user.firstName); // 'John' (dot notation)
console.log(user['lastName']); // 'Doe' (bracket notation)
console.log(user['has-admin-access']); // Bracket notation for special characters

// Calling a method
user.sayHello(); // "Hello, my name is John"

Iterating Over Objects

You can iterate over an object's properties using a `for...in` loop or by using `Object.keys()`, `Object.values()`, or `Object.entries()` to get an array of the object's keys, values, or key-value pairs, which you can then loop over.

// Using for...in (iterates over keys)
for (const key in user) {
  console.log(`${key}: ${user[key]}`);
}

// Using Object.entries() and for...of (modern approach)
for (const [key, value] of Object.entries(user)) {
  console.log(`${key}: ${value}`);
}

Sets

A **Set** is a collection of **unique** values. Each value can only occur once in a Set. Sets can hold any data type, from primitive values to object references.

Creating and Using Sets

You create a Set using the `new Set()` constructor. You can optionally pass an iterable (like an array) to initialize it.

  • .add(value): Adds a new element to the Set.
  • .has(value): Returns a boolean indicating whether an element exists in the Set.
  • .delete(value): Removes a specified element from the Set.
  • .clear(): Removes all elements from the Set.
  • .size: A property that returns the number of elements in the Set.
const mySet = new Set();

mySet.add(1);
mySet.add(5);
mySet.add('some text');
mySet.add(1); // This will be ignored, as 1 is already in the set

console.log(mySet); // Set(3) { 1, 5, 'some text' }
console.log(mySet.has(5)); // true
mySet.delete(5);
console.log(mySet.size); // 2

Use Case: Removing Duplicates from an Array

Sets provide a very concise way to get the unique values from an array.

const numbers = [2, 3, 4, 4, 2, 3, 3, 4, 4, 5];
const uniqueNumbers = [...new Set(numbers)];
console.log(uniqueNumbers); // [2, 3, 4, 5]

Maps

A **Map** is a collection of key-value pairs similar to an Object. However, a major difference is that a Map allows **keys of any data type** (including objects and functions), whereas an object's keys are limited to strings and Symbols. Maps also maintain the insertion order of their elements.

Creating and Using Maps

You create a Map using the `new Map()` constructor.

  • .set(key, value): Adds or updates an element with a specified key and value.
  • .get(key): Returns the value associated with the key, or `undefined` if the key doesn't exist.
  • .has(key): Returns a boolean indicating whether an element with the specified key exists.
  • .delete(key): Removes the element with the specified key.
  • .size: A property that returns the number of elements in the Map.
const myMap = new Map();

const keyString = 'a string';
const keyObj = {};
const keyFunc = function() {};

myMap.set(keyString, "value for a string");
myMap.set(keyObj, "value for an object");
myMap.set(keyFunc, "value for a function");

console.log(myMap.size); // 3
console.log(myMap.get(keyObj)); // "value for an object"

Objects vs. Maps

Feature Object Map
Key Types Strings and Symbols only. Any data type (string, number, object, function, etc.).
Order Not guaranteed (though modern engines often preserve it). Guaranteed insertion order.
Size Must be calculated manually (e.g., `Object.keys().length`). Directly available via the `.size` property.
Performance Historically optimized for property access. Optimized for frequent addition and removal of key-value pairs.
Use Case Good for simple, static collections of data (e.g., a config object). Better for dynamic collections, especially when keys aren't strings.
