JavaScript Regular Expressions (RegEx)

Introduction to RegEx

A **Regular Expression** (often shortened to RegEx or RegExp) is an object that describes a sequence of characters used for defining a search pattern. These patterns are used by string-searching algorithms to perform "find" or "find and replace" operations on strings, or for input validation.

In JavaScript, a regular expression is created using a literal, which consists of a pattern enclosed between slashes (`/`).

let regex = /pattern/flags;

Using RegEx with String Methods

Regular expressions are most powerful when used with string methods to perform searches and replacements.

  • search(regex): Searches a string for a match and returns the index of the first match. Returns -1 if no match is found.
  • match(regex): Retrieves the result of matching a string against a regular expression. Returns an array of matches or `null`.
  • replace(regex, replacement): Returns a new string where matched patterns are replaced with a specified string or the result of a function.
const str = "A long length of string with many long words.";

console.log(str.search(/long/)); // 2 (index of the first 'long')

console.log(str.match(/long/g)); // ['long', 'long']

console.log(str.replace(/long/gi, "short"));
// "A short length of string with many short words."

RegEx Flags

Flags (or modifiers) are placed after the second slash and change how the search is performed.

  • g (Global): Finds all matches in the string, not just the first one.
  • i (Case-Insensitive): Makes the matching case-insensitive (e.g., `/a/i` matches 'a' and 'A').
  • m (Multiline): Allows the start (^) and end ($) anchors to match the start and end of lines, not just the whole string.

Metacharacters (Character Classes)

Metacharacters are characters with a special meaning. They are used to create more specific and powerful patterns.

CharacterDescription
.Matches any single character except newline.
\dMatches any digit (equivalent to [0-9]).
\DMatches any non-digit.
\wMatches any word character (alphanumeric `[a-zA-Z0-9]` plus underscore `_`).
\WMatches any non-word character.
\sMatches a single whitespace character (space, tab, newline, etc.).
\SMatches a single non-whitespace character.
\bFinds a match at the beginning or end of a word (word boundary).
 
const text = "My phone number is 123-456-7890.";

// Find all digits
console.log(text.match(/\d/g).join('')); // "1234567890"

// Find all non-word characters
console.log(text.match(/\W/g)); // [' ', ' ', ' ', ' ', '-', '-', '.']

Brackets (Character Sets)

Brackets are used to find a range of characters.

  • [abc]: Finds any of the characters between the brackets.
  • [^abc]: The caret (^) negates the set, finding any character NOT between the brackets.
  • [0-9]: Finds any digit from 0 to 9.
  • [a-z]: Finds any lowercase character from a to z.
  • (a|b): Finds either 'a' or 'b'.
const str = "Is this gray or grey?";

// Find 'gray' or 'grey'
console.log(str.match(/gr(a|e)y/g)); // ["gray", "grey"]

// Find all vowels
console.log(str.match(/[aeiou]/gi)); // ['I', 'i', 'a', 'o', 'e']

Quantifiers

Quantifiers specify how many instances of a character, group, or character class must be present in the input for a match to be found.

QuantifierDescription
+Matches one or more occurrences of the preceding character.
*Matches zero or more occurrences.
?Matches zero or one occurrence.
{n}Matches exactly `n` occurrences.
{n,}Matches at least `n` occurrences.
{n,m}Matches from `n` to `m` occurrences.
 
const text = "hellooo world, helo";

// Matches 'o' two or more times
console.log(text.match(/o{2,}/g)); // ["ooo"]

// Matches 'helo' or 'hello'
console.log(text.match(/hel?o/g));   // null (needs to be /hel{1,2}o/ or similar)
// A better pattern for 'helo' or 'hello':
console.log(text.match(/hel{1,2}o/g)); // ["hellooo", "helo"] -> Corrected logic: /hello|helo/ or /hel+o/
console.log(text.match(/hel+o/g));   // ["hellooo", "helo"]
