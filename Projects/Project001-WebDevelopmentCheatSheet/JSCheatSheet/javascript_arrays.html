JavaScript Arrays

Definition

An array is a special type of object used for storing an ordered collection of multiple values in a single variable. The values can be of any data typeâ€”numbers, strings, objects, or even other arrays. Each value in an array is called an element, and each element is accessible via a numeric index, starting from zero.

Creating Arrays and Accessing Elements

The most common way to create an array is using an array literal (`[]`). You can also use the `new Array()` constructor.

// Array literal (preferred)
const fruits = ['Apple', 'Banana', 'Cherry'];

// Using the Array constructor
const numbers = new Array(10, 20, 30);

// Accessing elements using zero-based index
console.log(fruits[0]); // 'Apple'
console.log(fruits[2]); // 'Cherry'

// Modifying an element
fruits[1] = 'Blueberry';
console.log(fruits); // ['Apple', 'Blueberry', 'Cherry']

Array Length and Looping

The .length property returns the number of elements in an array. It's commonly used to iterate over the array with a for loop.

const colors = ['Red', 'Green', 'Blue', 'Yellow'];
console.log(colors.length); // 4

// Looping with a for loop
for (let i = 0; i < colors.length; i++) {
    console.log(colors[i]);
}

Adding/Removing Elements

JavaScript provides several methods to add or remove elements from an array.

  • .push(item): Adds one or more elements to the **end** of an array and returns the new length.
  • .pop(): Removes the **last** element from an array and returns that element.
  • .unshift(item): Adds one or more elements to the **beginning** of an array and returns the new length.
  • .shift(): Removes the **first** element from an array and returns that element.
  • .splice(start, deleteCount, item1, ...): A powerful method to change the contents of an array by removing, replacing, or adding elements **in place**.
  • .slice(start, end): Returns a **new array** containing a shallow copy of a portion of the original array. It does **not** modify the original array.

Difference Between .slice() and .splice()

This is a common source of confusion. The key difference is that .splice() modifies the original array, while .slice() does not.

Feature .slice() .splice()
Modifies Original Array? No (Immutable) Yes (Mutable)
Return Value A new array containing the extracted elements. An array containing the deleted elements.
Purpose To copy a portion of an array. To add, remove, or replace elements within an array. 
const animals = ['ant', 'bison', 'camel', 'duck', 'elephant'];

// Using slice()
const middleAnimals = animals.slice(2, 4);
console.log(middleAnimals); // ['camel', 'duck']
console.log(animals);       // Original array is unchanged

// Using splice()
const removed = animals.splice(1, 2, 'bear'); // Remove 2 elements from index 1 and add 'bear'
console.log(removed); // ['bison', 'camel']
console.log(animals); // ['ant', 'bear', 'duck', 'elephant'] - Original array is modified

Searching in Arrays

  • .indexOf(item) / .lastIndexOf(item): Return the first/last index of an element, or -1 if not found. Works well for primitive types.
  • .includes(item): Returns `true` or `false` if an element is found. More readable than `indexOf`.
  • .find(callback): Returns the **value** of the first element that passes a test provided by a callback function.
  • .findIndex(callback): Returns the **index** of the first element that passes a test.
  • .some(callback): Checks if at least **one** element passes the test. Returns `true` or `false`.
  • .every(callback): Checks if **all** elements pass the test. Returns `true` or `false`.

Transforming Arrays

These methods create **new arrays** based on the original, following immutable principles.

  • .map(callback): Creates a new array by calling a function on every element and storing the results.
  • .filter(callback): Creates a new array with all elements that pass the test in the callback function.
  • .reduce(callback, initialValue): Executes a "reducer" function on each element, resulting in a single output value (e.g., a sum, or a new object).
const numbers = [1, 2, 3, 4, 5];

const doubled = numbers.map(n => n * 2); // [2, 4, 6, 8, 10]
const evens = numbers.filter(n => n % 2 === 0); // [2, 4]
const sum = numbers.reduce((accumulator, current) => accumulator + current, 0); // 15

Sorting and Reversing

.sort() and .reverse() are **mutating** methods; they change the original array.

Important: By default, .sort() converts elements to strings and sorts them lexicographically. For numbers, you must provide a custom compare function.

const months = ['March', 'Jan', 'Feb', 'Dec'];
months.sort();
console.log(months); // ['Dec', 'Feb', 'Jan', 'March']

const numbers = [4, 2, 5, 1, 3, 10];
numbers.sort((a, b) => a - b); // Correct numeric sort
console.log(numbers); // [1, 2, 3, 4, 5, 10]

numbers.reverse();
console.log(numbers); // [10, 5, 4, 3, 2, 1]

Advanced Array Topics

Array.from() and Array.of()

  • Array.from(iterable): Creates a new array from an array-like or iterable object (like a NodeList from the DOM, a string, or a Set).
  • Array.of(...): Creates a new array from a variable number of arguments, regardless of type.
// Array.from a string
const helloArr = Array.from('hello');
console.log(helloArr); // ['h', 'e', 'l', 'l', 'o']

// Array.of
const stuff = Array.of(7, 'text', true);
console.log(stuff); // [7, 'text', true]

Empty Slots vs. undefined (Sparse Arrays)

A sparse array is one with "holes" or empty slots. These are not the same as elements with the value undefined. Most array iteration methods (like `map`, `filter`, `forEach`) **skip** empty slots entirely.

const sparse = [1, , 3];
const withUndefined = [1, undefined, 3];

console.log(sparse.length); // 3
sparse.forEach(val => console.log(val)); // Logs 1, then 3. Skips the empty slot.

withUndefined.forEach(val => console.log(val)); // Logs 1, undefined, then 3.

Arrays vs. Sets

A `Set` is a collection of **unique** values. Choose based on your needs:

  • Use an Array when order is important or you need to store duplicate values. Accessing by index (`arr[i]`) is very fast.
  • Use a Set when you need to store unique values and quickly check for the existence of a value (`set.has(item)`), which is much faster than an array's `includes()` for large collections.

Expert / Interview-Level Topics

Immutability with Arrays

Immutability means not changing data in place. In many modern frameworks (like React), it's a core principle. Instead of mutating an array, you create a new one with the desired changes.

Mutating Methods: push, pop, shift, unshift, splice, sort, reverse

Immutable Approach: Use methods that return new arrays like map, filter, slice, concat, and the spread operator ....

const users = [{id:1, name:'Alice'}, {id:2, name:'Bob'}];

// Immutable way to add a user
const newUser = {id:3, name:'Charlie'};
const updatedUsers = [...users, newUser];

// Immutable way to remove a user
const usersAfterRemoval = users.filter(user => user.id !== 1);

Performance Considerations

  • push() vs unshift(): Adding/removing from the end of an array (`push`/`pop`) is a very fast operation ($O(1)$). Adding/removing from the beginning (`unshift`/`shift`) is slow ($O(n)$) because every other element in the array must be re-indexed.
  • Choosing the right method: For searching, `Set.has()` is much faster than `Array.includes()` on large datasets. Use the right data structure for the job.
