Types of CSS Selectors

Here is a detailed breakdown of the most common types of selectors in CSS.

1. Element Selector (or Type Selector)

  • Definition: Selects all HTML elements based on their element name.
  • Detailed Information: This is the most basic selector. It targets every instance of a specific HTML tag on the page.
  • Syntax: element
  • Example: 
    p {
    color: blue;
}

2. Class Selector

  • Definition: Selects all elements that have a specific class attribute.
  • Detailed Information: Classes are used to apply the same style to multiple, different elements. Targeted with a period (.).
  • Syntax: .classname
  • Example: 
    .highlight {
    background-color: yellow;
}

3. ID Selector

  • Definition: Selects a single, unique element that has a specific id attribute.
  • Detailed Information: An id must be unique within an HTML document. Targeted with a hash (#).
  • Syntax: #idname
  • Example: 
    #header {
    background-color: #333;
}

4. Element-Specific Class Selector

  • Definition: Selects a specific element that also has a particular class.
  • Detailed Information: This is more specific than a general class selector. It allows you to style only certain types of elements that have a class, ignoring other elements that might share the same class. For example, p.highlight will only style paragraphs with the highlight class, not headings or divs with the same class.
  • Syntax: element.classname
  • Example: 
    p.highlight {
    border: 1px solid red;
}

5. Grouping Selector

  • Definition: Applies the same set of styles to multiple selectors by listing them in a comma-separated list.
  • Detailed Information: This is a way to avoid repeating code. If you have same styles that apply to h1, h2, and h3 elements, you can group them together instead of writing three separate rule sets.
  • Syntax: selector1, selector2, selector3
  • Example: 
    h1, h2, p {
    color: #444;
    font-family: Arial, sans-serif;
}

6. Descendant Selector

  • Definition: Selects an element that is located anywhere inside another element (a descendant).
  • Detailed Information: This selector targets nested elements. The selectors are separated by a space. For example, div p selects any p element that is inside a div, no matter how deeply it is nested (e.g., < div >< section >< p >...< /p >).
  • Syntax:ancestor descendant
  • Example: 
    .container p { /* id or element name can also be used */
  line-height: 1.5;
}

7. Child Selector

  • Definition:Selects an element that is a direct child of another element.
  • Detailed Information: Unlike the descendant selector, this is more strict. It only targets immediate children.
  • Syntax: parent > child
  • Example:
    #main-nav > li { /* class or element name can also be used */
    display: inline-block;
}

8. Adjacent Sibling Selector

  • Definition:Selects an element that is the very next sibling of another specified element.
  • Detailed Information:The two elements must have the same parent, and the second element must come immediately after the first one in the HTML source code.
  • Syntax:element1 + element2
  • Example:
    h2 + p {
    margin-top: 0;
}

9. General Sibling Selector

  • Definition: Selects all elements that are siblings of a specified element and appear after it in the HTML.
  • Detailed Information: his is less strict than the adjacent sibling selector. It selects all siblings that follow the first element, not just the one immediately after.
  • Syntax: element1 ~ element2
  • Example:
    h2 ~ p {
    text-indent: 2em;
}

10. Attribute Selector

  • Definition: Selects elements based on the presence or value of a given attribute.
  • Detailed Information:his is very powerful for targeting elements without needing a class or ID. You can select based on whether an attribute exists, its exact value, or even a partial value.
  • Syntax:
    • [attribute]- Selects elements with the specified attribute.
    • [attribute=value]- Selects elements where the attribute has an exact value.
    • [attribute~=value] - Selects elements where the attribute value is a space-separated list containing the given value.
    • [attribute|=value] - Selects elements where the attribute value starts with the given value in a hyphen-separated list (e.g., lang="en-US").
    • [attribute^=value] - Selects elements where the attribute value begins with the given value.
    • [attribute$=value] - Selects elements where the attribute value ends with the given value.
    • [attribute*=value] - Selects elements where the attribute value contains the given value.
  • Example:
    /* Selects all a tags with a 'target' attribute */
a[target] {
    background-color: powderblue;
}

/* Selects only a tags that open in a new tab */
a[target="_blank"] {
    font-weight: bold;
}

11. Pseudo-class Selector

  • Definition: Selects elements based on a special state, like being hovered over by the mouse, being the first child of its parent, or being a visited link.
  • Detailed Information: Pseudo-classes behave like a class that is dynamically applied by the browser in certain conditions. They are attached to a selector using a single colon
  • Syntax: selector:pseudo-class
  • Example:
    button:hover {
    background-color: #555;
    color: white;
}

12. Pseudo-element Selector

  • Definition: Styles a specific part of an element, rather than the whole element.
  • Detailed Information: Pseudo-elements allow you to style fragments of an element, such as the first line of a paragraph or to insert content before or after an element's actual content. They are attached to a selector using a double colon (::).
  • Syntax: selector::pseudo-element
  • Example:
    p::first-line {
    font-weight: bold;
    font-size: 1.2em;
}

Selector Differences

Descendant vs. Child Selector

Feature Descendant Selector (ancestor descendant) Child Selector (parent > child)
Symbol A single space ( ) A greater-than sign (>)
Relationship Selects any descendant, no matter how deeply nested. Selects only direct children (first-level).
Scope Broad and travels down the entire DOM tree from the ancestor. Narrow and strict, looks only at the next level.

Adjacent Sibling vs. General Sibling Selector

Adjacent Sibling (+): This selector is very specific. It only selects the single element that comes immediately after the former element. The two must be siblings (share the same parent). If there is any other element between them (even a line break that creates a text node), the selector will not match.

General Sibling (~): This selector is more flexible. It selects all sibling elements that come anywhere after the former element. They must share the same parent, but they don't have to be immediately next to each other.

Common Pseudo-class Selectors

  • :hover - Selects an element when the user mouses over it.
  • :active - Selects an element at the moment it is being clicked.
  • :focus - Selects an element that has keyboard focus.
  • :first-child - Selects an element that is the first child of its parent.
  • :last-child - Selects an element that is the last child of its parent.
  • :nth-child(n) - Selects an element based on its position among siblings.
  • :not(selector) - Selects every element that is not the specified selector.

Common Pseudo-element Selectors

  • ::before - Inserts generated content before an element's content.
  • ::after - Inserts generated content after an element's content.
  • ::first-line - Styles the first line of a block-level element.
  • ::first-letter - Styles the first letter of a block-level element.
  • ::selection - Styles the portion of text highlighted by the user.
  • ::placeholder - Styles the placeholder text in an input field.
