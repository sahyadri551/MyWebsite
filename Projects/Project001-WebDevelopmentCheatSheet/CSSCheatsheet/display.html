Display Property (Excluding Flex)

Definition

The display property is one of the most fundamental concepts in CSS. It specifies the type of rendering box used for an element, which determines how it behaves in the page layout (e.g., whether it starts on a new line, respects width/height, etc.).

Details (with Usage and properties)

Every HTML element has a default display value. For example, <div> and <p> are block-level elements, while <span> and <a> are inline. The display property allows you to override these defaults to achieve different layouts.

List of all possible CSS properties and their possible Values

  • display: Sets the element's inner and outer display types.
  • visibility: A related property that specifies whether an element is visible or hidden. Unlike display: none, a hidden element still takes up space in the layout.

Possible Values for `display` (common values, excluding flex)

  • block: The element generates a block-level box. It starts on a new line and stretches to fill the available width of its container. It respects width, height, margin, and padding.
  • inline: The element generates an inline-level box. It does not start on a new line and only takes up as much width as its content. It does not respect width, height, or vertical margin/padding.
  • inline-block: A hybrid of the two. The element flows along with text like an inline element, but it can have a set width and height and respects all margins and padding, just like a block element.
  • none: The element is completely removed from the document flow. It is not rendered and takes up no space.

Possible Values for `visibility`

  • visible (Default): The element is visible.
  • hidden: The element is invisible, but it still affects the layout by taking up its original space.

State difference between visibility:hidden and display:none with suitable illustration

This is a critical distinction in CSS. While both properties can make an element disappear, they do so in fundamentally different ways.

  • display: none; removes the element entirely from the page. The browser acts as if the element doesn't exist in the HTML, and it takes up no space. Other elements will shift to fill its void.
  • visibility: hidden; simply makes the element invisible. It's like painting it with transparent paint. The element is still there, occupying its original space in the layout, leaving a blank gap where it used to be.
