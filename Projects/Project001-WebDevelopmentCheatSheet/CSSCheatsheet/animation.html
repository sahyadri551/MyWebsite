CSS Animations

Definition

CSS Animations make it possible to animate transitions from one CSS style configuration to another. An animation consists of two components: a style describing the CSS animation and a set of keyframes that indicate the start and end states of the animation's style, as well as possible intermediate waypoints.

Details (with Usage and properties)

The power of CSS animations lies in the @keyframes at-rule. This rule allows you to define the stages and styles of the animation sequence. Once the keyframes are defined, you apply the animation to an element using the animation property (or its sub-properties).

  1. Define the animation's sequence using @keyframes.
  2. Apply the animation to an element using animation-name and animation-duration.
  3. Use other animation-* properties to customize timing, delay, iteration, etc.

Explain @keyframe

  • Definition: The @keyframes at-rule lets you specify the animation code. It creates a series of "stops" during the animation sequence.
  • Usage: Inside the @keyframes rule, you define styles for specific points in time using percentages (0% to 100%) or the keywords from (for 0%) and to (for 100%).

Example Keyframe Definition:

@keyframes fadeInAndSlide {
  from { /* or 0% */
    opacity: 0;
    transform: translateY(30px);
  }
  to { /* or 100% */
    opacity: 1;
    transform: translateY(0);
  }
}

List of all possible CSS properties

PropertyDescription
animation-nameSpecifies the name of the @keyframes at-rule.
animation-durationSpecifies how long one cycle of the animation takes.
animation-timing-functionSpecifies the speed curve of the animation (e.g., ease, linear).
animation-delaySpecifies a delay for the start of an animation.
animation-iteration-countSpecifies the number of times an animation should be played (e.g., 3, infinite).
animation-directionSpecifies whether an animation should play forwards, backwards, or in alternate cycles.
animation-fill-modeSpecifies what styles apply to the element when the animation is not playing.
animation-play-stateSpecifies whether an animation is running or paused.
animationA shorthand property for all the animation properties above.

Property Explanations and Illustrations

animation-name and animation-duration

  • Definition: animation-name specifies which @keyframes to use. animation-duration specifies how long the animation takes. Both are required for an animation to run.

Example:

I fade in and slide up over 2 seconds.

animation-iteration-count

  • Definition: Defines how many times the animation cycle should repeat.
  • Value: A number or infinite.

Example: animation-iteration-count: infinite;

I will spin and change color forever!

animation-direction

  • Definition: Sets whether an animation should play forwards, backwards, or alternate.
  • Value: normal, reverse, alternate, alternate-reverse.

Example: animation-direction: alternate;

The animation plays forwards, then backwards, and repeats.

I go up and down!

animation-play-state

  • Definition: Allows you to pause and resume an animation.
  • Value: running (default), paused.

Example: animation-play-state: paused; on hover

Hover over the box below to pause the animation.

Hover me to pause!

animation (Shorthand)

  • Definition: A shorthand property to set all animation properties in one declaration.

Example: animation: spinAndColor 4s ease-in-out infinite;

This box uses the shorthand property to configure its animation.

I am animated with shorthand!
