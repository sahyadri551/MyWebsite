HTML Media Elements

Definition

HTML media elements are used to embed media content like audio and video directly into a web page without requiring any external plugins. The primary tags for this are <audio> and <video>.

Details & Usage

The <audio> and <video> tags provide a native way for browsers to play media files. You can specify a single source file with the src attribute or provide multiple formats using the <source> tag to ensure cross-browser compatibility. It's crucial to always include the controls attribute to give users control over playback (play, pause, volume).

Media Elements and Attributes

  • <audio>: Embeds sound content.
    • src: URL of the audio file.
    • controls: Displays standard audio playback controls.
    • autoplay: Starts playing the audio automatically (often blocked by browsers).
    • muted: Mutes the audio by default. Often required for autoplay to work.
    • loop: Repeats the audio once it finishes.
    • preload: Specifies how the audio should be loaded ('none', 'metadata', 'auto').
  • <video>: Embeds video content.
    • All attributes from <audio> apply.
    • width & height: Sets the dimensions of the video player.
    • poster: URL of an image to display before the video starts playing.
  • <source>: Defines alternative media resources for <audio> or <video>.
    • src: URL of the media file.
    • type: The MIME type of the media resource (e.g., 'audio/mpeg', 'video/mp4').

Mailto Functionality of Anchor Tag

The anchor tag (<a>) can create a hyperlink that opens the user's default email client. This is done using the mailto: URI scheme in the href attribute. You can pre-fill the recipient's email address and optionally include a subject, body text, CC, and BCC. 

<!-- Basic mailto link -->
<a href="mailto:contact@example.com">Email Us</a>

<!-- Mailto link with subject and body -->
<a href="mailto:support@example.com?subject=Help%20Request&body=Please%20describe%20your%20issue.">
    Contact Support
</a>

Example Illustration

The code below demonstrates embedding a video with multiple sources and a fallback message.

<h3>Sample Video</h3>
<video controls width="600" poster="https://via.placeholder.com/600x340.png?text=Video+Poster">
  <source src="movie.mp4" type="video/mp4">
  <source src="movie.webm" type="video/webm">
  Your browser does not support the video tag.
</video>

<h3>Sample Audio</h3>
<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  Your browser does not support the audio element.
</audio>
