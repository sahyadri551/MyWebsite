HTML Forms

Definition

An HTML form is used to collect user input. The user input is most often sent to a server for processing. Forms are essential for creating interactive web pages, such as contact pages, login pages, and surveys.

Details & Usage

Forms are created using the <form> element, which acts as a container for different types of input elements, such as text fields, checkboxes, radio buttons, submit buttons, and more. The two most important attributes of the <form> tag are action (which specifies where to send the form data) and method (which specifies the HTTP method to use, typically GET or POST).

List of Form-related Tags

  • <form>: The container for a web form.
  • <input>: The most versatile form element; its behavior is defined by its type attribute.
  • <label>: Defines a label for an <input> element, improving accessibility.
  • <textarea>: Defines a multi-line text input control.
  • <button>: Defines a clickable button.
  • <select>: Creates a drop-down list.
  • <option>: Defines an option in a drop-down list.
  • <optgroup>: Groups related options in a drop-down list.
  • <fieldset>: Groups related elements in a form.
  • <legend>: Defines a caption for a <fieldset> element.
  • <datalist>: Specifies a list of pre-defined options for an <input> element.
  • <output>: Represents the result of a calculation.

Common Attributes

In addition to Global Attributes (like class, id, style, title), form elements have specific attributes:

  • for <form>:
    • action: URL where the form data is sent.
    • method: HTTP method (get or post).
    • enctype: Encoding type for form data, use multipart/form-data for file uploads.
    • target: Where to display the response (e.g., _blank, _self).
  • for <input>:
    • type: Specifies the type of input (e.g., text, password, checkbox, radio, submit, file, date, color).
    • name: The name of the input, sent to the server.
    • value: The initial value of the input.
    • placeholder: Hint text that disappears on focus.
    • required: Makes the input field mandatory.
    • disabled: Disables the input field.
  • for <textarea>:
    • rows: Specifies the visible number of lines.
    • cols: Specifies the visible width in characters.

Example Illustration

The code below demonstrates a simple contact form with various input types.

<form action="/submit-form" method="post">
    <fieldset>
        <legend>Contact Us</legend>
        <div class="form-group">
            <label for="name">Name:</label>
            <input type="text" id="name" name="user_name" required>
        </div>
        <div class="form-group">
            <label for="email">Email:</label>
            <input type="email" id="email" name="user_email" required>
        </div>
        <div class="form-group">
            <label for="inquiry">Reason for Inquiry:</label>
            <select id="inquiry" name="inquiry_reason">
                <option value="general">General Question</option>
                <option value="support">Technical Support</option>
                <option value="feedback">Feedback</option>
            </select>
        </div>
        <div class="form-group">
            <label for="message">Message:</label>
            <textarea id="message" name="user_message" rows="4"></textarea>
        </div>
        <div class="form-group">
            <input type="submit" value="Send Message">
        </div>
    </fieldset>
</form>
